Чек-лист для процессинг части проекта Minishell
Ниже представлен поэтапный план реализации процессинга в вашем Minishell. Следуя этому порядку, вы сможете постепенно усложнять функциональность и легче отлаживать каждую часть.

1. Базовое выполнение команд
 Запуск внешних программ:

Реализуйте функцию для запуска внешних команд с помощью fork() и execve(). - DONE
Передавайте аргументы и переменные окружения в execve().  - DONE
 Обработка статуса завершения:  - DONE

Отслеживайте код возврата дочернего процесса с помощью waitpid().  - DONE
Сохраняйте статус завершения для использования в дальнейшем. - DONE
2. Встроенные команды (built-ins) - DONE

cd: Изменение текущей директории.
echo: Вывод текста на стандартный вывод.
pwd: Вывод текущей директории.
export: Установка переменных окружения.
unset: Удаление переменных окружения.
env: Вывод всех переменных окружения.
exit: Завершение оболочки.

3. Перенаправления ввода/вывода
 Перенаправление вывода (> и >>):

Открывайте/создавайте файл для записи.
Перенаправляйте stdout в указанный файл с помощью dup2().
 Перенаправление ввода (<):

Открывайте файл для чтения.
Перенаправляйте stdin из указанного файла с помощью dup2().
 Heredoc (<<):

Реализуйте чтение до определенного ключевого слова.
Создавайте временный файл или используйте pipe для передачи данных.
4. Пайпы
 Реализация конвейеров (|):

Настройте пайпы между процессами с помощью pipe().
Перенаправляйте stdout первого процесса в stdin следующего с помощью dup2().
 Множественные пайпы:

Поддерживайте более двух команд в конвейере.
Управляйте несколькими пайпами и процессами.
5. Управление процессами
 Создание и управление процессами:

Используйте fork() для создания дочерних процессов.
Обрабатывайте зомби-процессы с помощью wait() или waitpid().
 Обработка фоновых процессов (опционально):

Реализуйте возможность запуска процессов в фоне (если требуется по спецификации).
6. Управление сигналами во время выполнения
Обработка сигналов:
Настройте обработчики сигналов для SIGINT, SIGQUIT и других, если необходимо.
Обеспечьте корректное поведение при нажатии Ctrl+C (обычно прерывает текущий процесс).
Игнорируйте Ctrl+D во время выполнения команд (обычно сигнал EOF в интерактивном режиме).
7. Переменные окружения
 Использование переменных окружения:

Обеспечьте доступ к переменным окружения внутри исполняемых команд.
 Изменение переменных окружения:

Реализуйте изменения в окружении через встроенные команды export и unset.
8. Обработка статусов завершения
 Сохранение статусов:

Сохраняйте код завершения последней выполненной команды в переменную ($?).
 Использование статусов:

Обеспечьте возможность использования кода возврата в скриптах или для отображения.
9. Дополнительные требования
 Поддержка комбинаций перенаправлений и пайпов:

Обеспечьте корректную работу при комбинации пайпов и перенаправлений в одной командной строке.
 Управление файлами дескрипторов:

Закрывайте ненужные файловые дескрипторы во избежание утечек.
 Обработка ошибок:

Выводите понятные сообщения об ошибках при сбоях системных вызовов.
Обрабатывайте случаи, когда команды или файлы не найдены.
10. Тестирование и отладка
 Создание тестовых сценариев:

Напишите набор тестов для проверки каждой реализованной функции.
 Отладка:

Используйте инструменты отладки для отслеживания проблем.
Проверяйте утечки памяти с помощью valgrind или аналогичных инструментов.
Рекомендации по порядку реализации:

Базовое выполнение команд и встроенные команды:

Начните с возможности запускать внешние программы и реализуйте основные встроенные команды. Это создаст основу вашей оболочки.
Перенаправления ввода/вывода:

Добавьте функциональность перенаправления, чтобы пользователи могли перенаправлять вывод и ввод команд.
Пайпы:

Реализуйте пайпы для поддержки конвейерной обработки команд. Это сложнее, так что лучше делать это после освоения перенаправлений.
Управление процессами:

Убедитесь, что вы правильно создаете и завершаете процессы, избегая зомби-процессов.
Управление сигналами:

Добавьте обработку сигналов, чтобы ваша оболочка корректно реагировала на прерывания.
Переменные окружения:

Реализуйте возможность изменять и использовать переменные окружения через export и unset.
Обработка статусов завершения:

Обеспечьте правильное сохранение и использование кодов возврата команд.
Дополнительные функции и отладка:

После реализации основных функций, добавьте поддержку сложных комбинаций и тщательно протестируйте оболочку.
